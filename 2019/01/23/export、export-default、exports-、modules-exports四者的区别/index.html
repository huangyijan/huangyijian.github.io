export、export default、exports 、modules.exports四者的区别

之前一直都对这四个导出模块有疑问，今天特地查了一下，做个小归纳

四者其实分为两大类(node和es6)

  • node分类： exports和module.exports(两个尾部带有s)的属于node的api文档上面module模块的内容，node文档直通道,exports是module.exports的简短的引用形式（官方原话：A reference to the module.exports that is shorter to type.），两者的区别在这：区别,但是在具体的使用中使用exports就ok了，以后开发对node熟悉了可以再理解一下两者的区别
  • es6分类 export和export default，先贴个阮一峰的es6文档，es6的module模块。这里面写的挺清楚了，export default 这里的default其实是一个默认内置的变量，使用者在import可以为这个值重命名，其实就是给人偷懒用的，export 使用就写好具体的导出名，在import的时候要使用"{导出名}"这种形式来使用。
import {firstName, lastName, year} from './profile.js';
